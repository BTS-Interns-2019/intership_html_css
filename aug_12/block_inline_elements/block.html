Block elements

    <address>

    address element indicates that the HTML provides a contact information for a person or people, or for organization.

    <article>

    article element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.

    A given document can have multiple articles in it; for example, on a blog that shows the text of each article one after another as the reader scrolls.

    <aside>

    aside element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.

    <blockquote>

    blockquote element (or HTML Block Quotation Element) indicates that the enclosed text is an extended quotation.

    <details>

    details element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. A summary or label can be provided using the summary element .

    <dialog>

    dialog element represents a dialog box or other interactive component, such as an inspector or window.

    <dd>

    dd element provides the description, definition, or value for the preceding term (dt) in a description list (dl).

    <div>

    div element is the generic container for content.

    <dl>

    dl elementrepresents a description list. This element encloses a list of groups of terms. Uses the element dt for titles and dd for descriptions

    <dt>

    dt element specifies a term in a description or definition list, it must be inside a dl element. Is usually followed by dd element.

    <fieldset>

    fieldset elementis used to group several controls as labels within web form

    figcaption>

    figcaption element represents a simple caption to describe the content of his parent element figure

    <figure>

    figure element represents a self-contained content, potentially with an optional caption

    <footer>

    footer element represents a sectioning content. Usually contains information about the author, copyright data, etc.

    <form>

    form element represents a document section tha contains inputs and tags for submitting information to the web server

    h1, h2, h3, h4, h5, h6 tags

    h1-h6 elements represents the different levels of sectioning in HTML, From the highest (<h1>) to the lowest (<h6>).

    <header>

    header element represents introductory contains. It may contains some heading elements but also a logo, a search form, an author name, etc.

    <hgroup>

    hgroup element encloses headers, represents a section made with headers from <h1> to <h6>

    <hr>

    hr element represents a break line between paragraphs. This could divide to sections.

    <li>

    li element represents an item in a list. It must be contained in an ol tag (ordered list), an ul tag (unordered list) or menu tag

    <main>

    main element just represents the domain content of the body of a document. This the most important information, the central topic, or the central functionality of an application

    <nav>

    nav element represents a section tha just provide navigation links

    <ol>

    ol element represents a ordered list of items

    <p>

    p element represents a paragraph, a text block

    <pre>

    pre element represents preformatted text which is to be presented exactly as written in the HTML file

    <section>

    section element represents a section of a document. Just for organization

    <table>

    table element represents a table data. The information os represented in a two-dimensional table

    <ul>

    ul element represent a list with unordered items
