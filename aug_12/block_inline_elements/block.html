Block elements

The Block elements occupies full space or full widht of its parent element or body, so block elements will appear in a new line each one of them.

Here a list of Block elements:

  • The paragraph element < p >:

    • Appropiate to distribute text to paragraphs. Also known as Phrase Element. It is mandatory both Open - Close tags.

  • The address element < address >:

    • Child element of < article > elements. Address element aports content to article element. It is mandatory a open and close tag.

  • The article element < article >:

    • Represent a self-content composition in a document, web page or aplication that self-distributes independently. Some of the elements that we find commonly in article element are < time > or < address >.

  • The aside element < aside >:

    • Section of a web page that is indirectly related to the principal content of the page. Commonly representated as lateral bars, insertions on margin as a glossary, publicity tabs, author's bio, profile info, etc. Self-closing tag.

  • The audio element < audio > :

    • Used to insert audio content to an HTML or XHTML document. Implemented in last version of HTML (HTML5). Self-closing tag.

  • The blockquote element < blockquote > :

    • Create quotes in block display of other authors or documents. Need both Open - Close tags.

  • The canvas element < canvas > :

    • Commonly related to Javascript manage. Used to graphic design through sequenced commands. Self-closing tag.

  • The dd element < dd > :

    • Commonly nested to < dl > tag. Specifically dd is used to set the description and details of definition of a preceeded term. Need of both Open-Close tags.

  • The details element < details > :

    • Shows a widget dropdown. Nested tag < summary > inside.

  • The dl element < dl > :

    • Represent a decriptive list. This list commonly encloses a list of terminologies specified on nested tag < dt >. Need of both Open-Close tags.

  • The div element < div > :

    • Used to create other sub-blocks in the body, also to group them properly. Needs of both Open-Close tags.

  • the fieldset element < fieldset > :

    • Order in groups those fields from a formulary. Both Open-Close tags needed.

  • The figcaption element < figcaption > :

    • Nested to parent element < figure >. figcaption content the subtitle or leyend asociated to content on figure element. Self Open-Close tag. Implemented on HTML5.

  • The figure element < figure > :

    • Mainly an image, diagram, ilustration or fragment of code that is referenced in main content of the page. figue element flows independently. Self-closing tag.

  • The footer element < footer > :

    • Commonly nested in article, aside, nav, section, blackquote, body, details, fieldset, figure, td elements. A footer mainly contents some author's information, links to related documents and the copy right data.

  • The form element < form > :

    • Interactive section of a page that let client send information to a server.

  • The header element < header > :

    • Represent help or guidance to navigation. May content header logo.

  • The hgroup element < hgroup > :

    • May attach a entire group of 'headers'. Nested elements can be any of this: h1, h2, h3, h4, h5, h6.

  • The hr element < hr > :

    • Applied to texts. Remaks a change of subject between paragraphs. It is represented by a full-widht line.

  • The li element < li > :

    • Explained version is 'item list', declares each element of a list specified by default by a dot to each item.

  • The main element < main > :

    • Represent the main content of the body or document. Shouln't be more than one main element or be nested in an article, aside, footer, etc elements.
    If extra information is needed will be set aside elements or else.

  • The nav element < nav > :

    • Represent a section that provides navigation links inside document or external documents.

  • The noscript element < noscript > :

    • Gives an alternative for scripts not supported for some browsers versions. Insead shows only content.

  • The ol element < ol > :

    • Creates an ordered list.

  • The pre element < pre > :

    • Shows content in exclusive format of the original file. With no special proportion of browser.

  • The section element < section > :

    • Represent a generic section of a document.

  • The table element < table > :

    • Represent data in two or more dimentions.

  • The tfoot element < tfoot > :

  • The ul element < ul > :

    • Creates a not ordered list.

  • The block element < block > :

    • Serves to inject video files to HTML or XHTML documents.
