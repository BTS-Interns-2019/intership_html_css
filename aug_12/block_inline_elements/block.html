Block Elements

Address Tag

Indicates that the enclosed HTML provides contact information for a person or people, or for an organization.

The contact information can take whatever form is appropriate for the context, and may include any type of contact information that is needed, such as physical address, URL, email address, phone number, social media handle, geographic coordinates, and so forth.The address element should include the name of the person, people, or organization to which the contact information refers.

Element to describe data contact for a person or company

Article Tag

Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable. Examples include: a forum post, a magazine or newspaper article, or a blog entry.

A given document can have multiple articles in it; for example, on a blog that shows the text of each article one after another as the reader scrolls, each post would be contained in an element.

Element to group articles in a html document

Aside Tag

Represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.

Element to add aditional informacion to an article

Blockquote Tag

Indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation.

Usado para citar

Details Tag

Creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state.

A disclosure widget is typically presented onscreen using a small triangle which rotates (or twists) to indicate open/closed status, with a label next to the triangle.

Usado para detallar algun punto o tema en el documento.

Dialog Tag

Represents a dialog box or other interactive component, such as an inspector or window.

Ventana de dialogo

DD Tag

DD (Description Details) provides the description, definition, or value for the preceding term (dt) in a description list (dl).

Definicion de un Termino en una lista

Div Tag

Is the generic container for flow content. It has no effect on the content or layout until styled using CSS.

It's used to group content so it can be easily styled using the class or id attributes, marking a section of a document as being written in a different language (using the lang attribute), and so on.

Usado para agrupar elementos y aplicarles cierto diseño basandose en su ID.

DL Tag

DL (Description List) represents a description list. The element encloses a list of groups of terms (specified using the dt element) and descriptions (provided by dd elements).

Lista para definir terminos.

DT Tag

DT (Description Term)represents a description list. The element encloses a list of groups of terms (specified using the dt element) and descriptions (provided by dd elements).

Cada termino de la lista.

Fieldset Tag

Is used to group several controls as well as labels within a web form.

Figcaption Tag

The HTML figcaption or Figure Caption element represents a caption or legend describing the rest of the contents of its parent figure element.

Pie de Foto.

Figure Tag

The HTML figure (Figure With Optional Caption) element represents self-contained content, potentially with an optional caption, which is specified using the (figcaption) element. The figure, its caption, and its contents are referenced as a single unit.

Sirve para insertar una foto en el.

Footer Tag

The HTML footer element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.

Sirve para ponerse al pie del documento.

Form Tag

The HTML form element represents a document section that contains interactive controls for submitting information to a web server.

Usado para elaborar un formulario.

Headings Tags

Heading levels 1-6. The HTML h1–h6 elements represent six levels of section headings. h1 is the highest section level and h6 is the lowest.

Son usados para los titulos y subtitulos en una pagina.

Header Tag

The HTML header element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.

Es la parte superior del documento y por lo general es usada para proporcionar datos sobre el titulo y los metadatos, no para mostrar contenido de la pagina.

Hgroup Tag

The HTML hgroup element represents a multi-level heading for a section of a document. It groups a set of h1–h6 elements.

Sirve para agrupar los headings

HR Tag

Horizontal rule (dividing line).The HTML hr element represents a thematic break between paragraph-level elements: for example, a change of scene in a story, or a shift of topic within a section.

Es una linea que delimita contenidos en el documento.

LI Tag

List item. The HTML li element is used to represent an item in a list. It must be contained in a parent element: an ordered list (ol), an unordered list (ul), or a menu (menu). In menus and unordered lists, list items are usually displayed using bullet points. In ordered lists, they are usually displayed with an ascending counter on the left, such as a number or letter.

Usado para representar cada elemento en una lista.

Main Tag

Contains the central content unique to this document. The HTML main element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.

Contiene el contenido principal de la pagina.

Nav Tag

Contains navigation links. Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.

Usada para los elementos de navegación.

OL Tag

Ordered List. Represents an ordered list of items, typically rendered as a numbered list.

Usado para declarar una nueva lista de forma ordenada.

P Tag

Represents a paragraph. Paragraphs are usually represented in visual media as blocks of text separated from adjacent blocks by blank lines and/or first-line indentation, but HTML paragraphs can be any structural grouping of related content, such as images or form fields.

Es un parrafo en el documento.

Pre Tag

Represents preformatted text which is to be presented exactly as written in the HTML file. The text is typically rendered using a non-proportional ("monospace") font. Whitespace inside this element is displayed as written.

Pasara el texto al documento tal y como se escribio.

Section Tag

Represents a standalone section — which doesn't have a more specific semantic element to represent it — contained within an HTML document. Typically, but not always, sections have a heading.

Parecido al div utilizado para seccionar el documento.

Table Tag

Represents tabular data — that is, information presented in a two-dimensional table comprised of rows and columns of cells containing data.

Usado basicamente para elaborar tablas.

UL Tag

Unordered list. Represents an unordered list of items, typically rendered as a bulleted list.

Usado para declarar una nueva lista con elementos no enumerados.
