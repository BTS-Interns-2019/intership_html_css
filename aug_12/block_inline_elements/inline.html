  2. " abbr>"

  3. " acronym>"

    allows authors to clearly indicate a sequence of characters that compose an acronym or abbreviation for a word. This element has been removed in HTML5. Use  element."><acronym>

  4. " audio>"

    element is used to embed sound content in documents. It may contain one or more audio sources, represented using the src attribute or the element: the browser will choose the most suitable one. It can also be the destination for streamed media, using a MediaStream."><audio> (if it has visible controls)

  5. " b>"

  6. " bdi >"

      tells the browser's bidirectional algorithm to treat the text it contains in isolation from its surrounding text."><bdi>

  7. " bdo>"

    overrides the current directionality of text, so that the text within is rendered in a different direction."><bdo>

  8. " big>"

    renders the enclosed text at a font size one level larger than the surrounding text (medium becomes large, for example)."><big>

  9. " br>"

    element produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant."><br>

  10. " button>"

    element represents a clickable button, which can be used in forms or anywhere in a document that needs simple, standard button functionality."><button>

  11. " canvas>"

    element with either the canvas scripting API or the WebGL API to draw graphics and animations."><canvas>

  12. " cite>"

    is used to describe a reference to a cited creative work, and must include the title of that work."><cite>

  13. " code>"

    element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code."><code>

  14. " data>"

    element links a given content with a machine-readable translation. If the content is time- or date-related, the element must be used."><data>

  16. "del"

    element represents a range of text that has been deleted from a document."><del>

  17. "dfn"

    is used to indicate the term being defined within the context of a definition phrase or sentence."><dfn>

  18. "em"

    element marks text that has stress emphasis. The element can be nested, with each level of nesting indicating a greater degree of emphasis."><em>

  19. "embed"

    element embeds external content at the specified point in the document. This content is provided by an external application or other source of interactive content such as a browser plug-in."><embed>

  20. "i"

    element represents a range of text that is set off from the normal text for some reason. Some examples include technical terms, foreign language phrases, or fictional character thoughts. It is typically displayed in italic type."><i>

  21. "iframe"

    represents a nested browsing context, embedding another HTML page into the current one."><iframe>

  22. "img"

    element embeds an image into the document. It is a replaced element."><img>

  23. "input"

    element is used to create interactive controls for web-based forms in order to accept data from the user; a wide variety of types of input data and control widgets are available, depending on the device and user agent."><input>

  24. "ins"

    element represents a range of text that has been added to a document."><ins>

  25. "kbd"

    represents a span of inline text denoting textual user input from a keyboard, voice input, or any other text entry device."><kbd>

  26. "label"

    element represents a caption for an item in a user interface."><label>

  27. "map"

    element is used with elements to define an image map (a clickable link area)."><map>

  28. "mark"

    represents text which is marked or highlighted for reference or notation purposes, due to the marked passage's relevance or importance in the enclosing context."><mark>

  29. "meter"

    element represents either a scalar value within a known range or a fractional value."><meter>

  30. "noscript"

    element defines a section of HTML to be inserted if a script type on the page is unsupported or if scripting is currently turned off in the browser."><noscript>

  31. "object"

    element represents an external resource, which can be treated as an image, a nested browsing context, or a resource to be handled by a plugin."><object>

  32. "output"

    is a container element into which a site or app can inject the results of a calculation or the outcome of a user action."><output>

  33. "picture"

    element contains zero or more elements and one element to provide versions of an image for different display/device scenarios."><picture>

  34. "progress"

    element displays an indicator showing the completion progress of a task, typically displayed as a progress bar."><progress>

  35. "q"

    element indicates that the enclosed text is a short inline quotation. Most modern browsers implement this by surrounding the text in quotation marks. "><q>

  36. "ruby"

    element represents a ruby annotation. Ruby annotations are for showing pronunciation of East Asian characters."><ruby>

  37. "s"

    element renders text with a strikethrough, or a line through it. Use the element to represent things that are no longer relevant or no longer accurate. However, is not appropriate when indicating document edits; for that, use the and elements, as appropriate."><s>

  38. "samp"

    is used to enclose inline text which represents sample (or quoted) output from a computer program."><samp>

  39. "script"

    element is used to embed or reference executable code; this is typically used to embed or refer to JavaScript code."><script>

  40. "select"

    element represents a control that provides a menu of options"><select>

  41. "slot"

    element—part of the Web Components technology suite—is a placeholder inside a web component that you can fill with your own markup, which lets you create separate DOM trees and present them together."><slot>

  42. " small "

    element makes the text font size one size smaller (for example, from large to medium, or from small to x-small) down to the browser's minimum font size.  In HTML5, this element is repurposed to represent side-comments and small print, including copyright and legal text, independent of its styled presentation."><small>

  43. " span "

    element is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang."><span>

  44. " strong "

    indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type."><strong>

  45. " sub"

    specifies inline text which should be displayed as subscript for solely typographical reasons."><sub>

  46. " sup"

    specifies inline text which is to be displayed as superscript for solely typographical reasons."><sup>

  47. "template"

    element is a mechanism for holding client-side content that is not to be rendered when a page is loaded but may subsequently be instantiated during runtime using JavaScript."><template>

  48. "textarea"

    element represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text, for example a comment on a review or feedback form."><textarea>

  49. "time"

    element represents a specific period in time."><time>

  50. " u"

    represents a span of inline text which should be rendered in a way that indicates that it has a non-textual annotation."><u>

  51. " tt"

    creates inline text which is presented using the user agent's default monospace font face."><tt>

  52. represents the name of a variable in a mathematical expression or a programming context."><var>

  53. "video"

    embeds a media player which supports video playback into the document."><video>

  54. "wbr"

    element represents a word break opportunity—a position within text where the browser may optionally break a line, though its line-breaking rules would not otherwise create a break at that location."><wbr>
