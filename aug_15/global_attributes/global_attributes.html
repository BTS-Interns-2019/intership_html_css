Global attributes

The Global attributes are shared in every HTML element, but not every attribute may cause effect the element it self. Also,
any other non-standard attribute must implementate them direct or undirectly due HTML5.

Here a list and description of Global attributes:

The accesskey attribute
Provides a keyboard shortcur for designed element.
The autocapitalize attribute.
Controls which way the test input will be automaticlly capitalized.
It may have next values:
  • off or none ; default all chars to lowercase.
    •
  • no or sentences ; the first letter of each sentence UpperCased.
  • word ; the first letter of each word Uppercased.
  • character ; all characters default to UpperCase.
    • The class attribute.
    List of classes of the elements. The class attribute let CSS and JS make use of the 'class'ed element.
    The contenteditable attribute.
    Enumerated attribute indicating if the element should be editable by the user. It takes the next two values:
  • true or editable ; declares that the element must be editable.
  • false ; declares that the element must not be editable.
    • The contextmenu attribute.
    The id of a < menu > to use as the contextual menu for this element.
    The data-* attribute.
    Forms a class of attributes called custom data attributes that allows that propetary information be exchanged
    between HTML and DOM interpretation from scripts
    The dir attribute.
    Declares direction of the text of the element. May contain next values:
  • ltr ; means left to right, used for left-to-right written languages.
  • rtl ; means right to left, used for right-to-left written languages.
  • auto ; gives desition of which ltr or rtl to agent. Uses a basic algorithm that analize characters inside element.
    • The draggable attribute.
    An enumerated attribute indicating if element can be dragged. May contain next values:
  • true ; declares that element may be dragged.
  • false ; declares that element can't be dragged.
    • The dropzone attribute.
    A enumerated attribute indicating if element can be dropped. May contain next values:
  • copy ; indicates that dropping an element will create a copy of element that was dragged.
  • move ; indicates that dropping an element will change place of element to designed place.
  • link ; will create a data of dragged data.
    • The hidden attribute.
    A Boolean attribute indicates if element isn't yet or no longer relevant. Browser wond render
    such element until a defined proccess is complete.
    The id attribute.
    Defines an unique identifier with a keyword so element can be accessed on CSS or JS.
    The inputmode attribute.
    Provides a hint to browsers as to the type of wirtual keyboard configuration in case of editing element's content. Primarly used in < input >.
    The is attribute.
    Allow to specify that standard HTML elements should behave like custom register.
    The itemid attribute.
    Unique, global identifier of an item.
    The itemprop attribute.
    Used to add properties to an item.
    The itemref attribute.
    Properties that are not descendants of an element with the itemscope attribute can be associated with the item using itemref.
    The itemscope attribute.
    Works with itemtype to specify that the HTML block is about an specific item. Creates the item and defines itemtype association.
    The itemtype attribute.
    Defines the URL of vocabulary that will be used to define data structure. itemscope define the data structure to specific HTML block.
    The lang attribute.
    Defines language of an specific element. Specific language of the element is non-editable. Editable elements should be written by user.
    The slot attribute.
    Assign a slot in a shadow DOM shadow tree to an element: An element with a slo attribute is assigned to the slot
    created by the < slot > element whose name attribute value matches that slot.
    The spellcheck attribute.
    Defines if an element will be checked out for spelling error. May contain next values:
  • true ; which indicates that element must be checked out for possible spelling errors.
  • false ; which indicates that element must not be checkout out for possible spelling errors.
    • The tabindex attribute.
    An integer attribute indicating if element take focus or is focusable.
  • negative value ; declares that element should be focusable but not reachable bia sequantial keyboard navigation.
  • 0 ; means that element should be focusable and reachable via sequential keyboard navigation
  • positive value ;
    • The title attribute.
    Contains information of the elements title belongs to.
    The translate attribute.
    Enumerated attribute that is used to specify whether an element's attribute values and the values of its Text node children are
    to be translated when the page is localized. May contain next vaules:
  • empty string and yes ; that specifies that element will be translated.
  • no ; indicates that element won't be translated.
