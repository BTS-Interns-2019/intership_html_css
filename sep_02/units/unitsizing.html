CSS Size Units

There are many data types to use while setting values to CSS attributes. Those are:

  • < integer >
  • < number >
  • < dimension >
  • < percentage >

Absolute length units:

Unit Name Equivalent
cm Centimiters 1cm = 96px / 2.54
mm Millimeters 1mm = 1/ 10th of 1 cm
Q Quarter-millimeters 1Q = 1 / 40th of 1cm
in Inches 1in = 2.54cm = 96px
Pc Picas 1pc = 1 / 16th of 1in
pt Points 1pt = 1 / 72th of 1in
px Pixels 1px = 1 / 96th of 1in

Relative length Units

Unit Relative to
em Font size of the parent element.
ex x-height of the element's font.
rem Font size of the root element.
lh Line height of the element.
vw 1% of the viewpor's width.
vh 1% of he viewpor's height.
vmin 1% of the viewport's smaller dimensions.
vmax 1% of the viewport's larger dimension.

About...

There is some interesting stuff so relative to take in count about measures, specially in percentage. The element with a percentage designated will take
as it's 100% the father's element width.

The same thing happens with units em and rem but inverse. Attributes will take size reference of father element to design its
its size.

Color and Hexadecimals

There are three type of values that can set a color. The human readable string that declares the color like 'red', 'black', the hexadecimal unit like
#000000 for black, #FF0000 for red, #00FF00 for green and #0000FF for blue and a mixture that is declaring the rgb
'method', that takes three parameteres to return the color desired. : rgb(15,15,15)...
